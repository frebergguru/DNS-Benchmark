# Internationalization

Standard GNU gettext. Sources are English; runtime translations come
from `po/*.po` compiled to `share/locale/<lang>/LC_MESSAGES/dnsbenchmark.mo`.

## Adding a new language

1. Run `msginit` to seed the file:

   ```bash
   cd po
   msginit --locale=fr_FR.UTF-8 --input=dnsbenchmark.pot --output=fr.po
   ```

2. Translate the `msgstr` lines in `fr.po`.

3. Reconfigure & rebuild:

   ```bash
   cmake -S . -B build
   cmake --build build -j
   ```

   The `translations` target re-globs `po/*.po` and compiles each one to
   `<build>/locale/<lang>/LC_MESSAGES/dnsbenchmark.mo`. CMake installs
   them under `share/locale/<lang>/LC_MESSAGES/`.

4. Add the language to the popover in `build_lang_button` (`window.c`)
   if you want it as a one-click choice — otherwise it works
   automatically when the user's `LANG` matches.

## Updating after source changes

When you add new `_()` calls in C, regenerate the template:

```bash
xgettext --keyword=_ --keyword=N_ --from-code=UTF-8 \
  --copyright-holder='dnsbenchmark contributors' \
  --package-name=dnsbenchmark --package-version=0.1.0 \
  --output=po/dnsbenchmark.pot \
  $(find src -name '*.c')
```

Then merge new strings into existing `.po` files:

```bash
msgmerge --update po/nb.po po/dnsbenchmark.pot
```

Translate the new entries, then `msgfmt --check po/nb.po` to validate.

## Conventions

- **Mark every user-visible string with `_("...")`.** Don't concatenate
  pre- and post-translation; let the whole sentence be one msgid so
  translators can reorder words.
- Use `%s`, `%d`, `%zu` directly in the msgid. For multi-arg formats
  prefer positional `%1$s %2$d` if Norwegian needs a different order.
- Log messages (`DNSB_INFO`, `DNSB_WARN`, `DNSB_ERROR`) stay in
  English — they're for developers / sysadmins.

## How runtime switching works

Gettext caches translations keyed by domain + locale + msgid. Once a
process has resolved `_("Run")` in English, asking again returns the
cached "Run" even after `setlocale` changes. The standard escape hatch
is to bump glibc's internal `_nl_msg_cat_cntr` — `dnsb_i18n_set_language`
in `src/util/i18n.c` does this.

That invalidates the cache for future calls but **does not** re-translate
already-rendered widgets — those hold strings in their internal state.
So the language picker also calls `dnsb_app_rebuild_window`, which
destroys the current window and creates a fresh one. The engine pointer
and all recorded stats are preserved across the swap (a live benchmark
is stopped first to avoid worker threads racing the rebuild).

## Default-language resolution

```
1. Persisted preference (~/.config/dnsbenchmark/lang)         ← highest
2. LANGUAGE env var (gettext-specific override)
3. LANG / LC_MESSAGES / LC_ALL env vars (POSIX setlocale)     ← system
```

Choosing **Auto** in the popover clears the persisted preference and
unsets `LANGUAGE`, falling through to the system locale.

## Files

| File                          | Purpose                                       |
|-------------------------------|-----------------------------------------------|
| `src/util/i18n.{c,h}`         | Init, preference load/save, runtime switch.   |
| `po/dnsbenchmark.pot`         | Template (regenerated by `xgettext`).         |
| `po/nb.po`                    | Norwegian Bokmål translation.                  |
| `build/locale/<lang>/LC_MESSAGES/dnsbenchmark.mo` | Compiled, runtime-loaded by gettext. |
